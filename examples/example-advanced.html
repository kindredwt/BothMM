BothMM.js

What's all this then?

This is an example/demo page for the BothMM.js library which allows using both unicode and Zawgyi-One (and a couple of other) encodings on the same page and recode it for the particular font installed in the user's browser.

This page has Burmese text written in different encodings and each block is marked up with a both-mm="..." attribute to indicate which encoding is used. Supported encodings are:

  • unicode for the international standard and universal encoding.
  • zawgyi-one for the most common legacy encoding.
  • wininnwa (Win Researcher, Win Innwa, Win Innwa, Win---Innwa, Win---Researcher)
  • wwin_burmese1 (Wwin_Burmese)
  • off to disable transcoding for a sub-block

Right below here is some information about the library and the state of the transforms. Below that are buttons to select the encoding to use to display the text.

Finally, the reason for all of this, there are boxes with the same text written in different source encodings. Selecting an encoding causes the text in all the boxes to be transcribed to the selected encoding. Selecting [off] will show all the texts in their original form. The transcribed text is also coloured when it is transcribed.

This page is comewhat complex with a big scary <script> block at the bottom, which is there to show some of the more complex functions of the library. There is also a simpler version of this page which works without any scripting.
